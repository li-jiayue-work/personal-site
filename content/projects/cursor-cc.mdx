## 工作区是什么

一个 monorepo 结构的工作区，`active/` 放活跃项目、`archive/` 存已完成项目、`agents/` 管 Agent 和 Workflow、`memory/` 做跨会话经验沉淀——共 9 个目录按职责划分。根目录 `CLAUDE.md` 定义全局规范，在任何子目录下执行 Claude Code 时自动加载上下文。

## 产出了什么

- **个人网站** — Next.js 16 + Tailwind CSS v4，MDX 驱动，中英双语
- **亚马逊数据分析项目** — 两个完整数据挖掘项目（产品 + 图书），Python 全流程
- **30+ HTML 模板库** — 标准化模板结构，含截图与构建脚本
- **Workflow / Skill / Memory 方法论体系** — 标准化流程 + 可复用技能模块 + 经验沉淀机制

## 核心收获

- **如何写让 AI 可执行的指令** — 上下文质量决定输出质量，CLAUDE.md 和 Memory 的投入是 ROI 最高的
- **如何在 AI 辅助下保持项目控制力** — 外科手术式修改、Reviewer 强制检查、最小修改原则
- **如何把工作流标准化** — 把每次成功的协作模式固化为 Workflow，从"靠运气"变成"靠流程"
